# JETNET Insight ‚Äî Field ‚Üí UI Presentation Mapping

### (‚ÄúHow it should appear in the UI‚Äù)

This converts the raw API fields into **interaction primitives**.
Goal: users never feel like they are reading database output ‚Äî they are *understanding an aircraft instantly*.

UI components referenced below:

* **Hero** = top aircraft header
* **Chip** = small contextual label
* **Badge** = colored status indicator
* **Card Row** = key-value in summary card
* **Timeline Event** = chronological marker
* **Capability Tag** = grouped functional feature
* **Insight Metric** = highlighted numeric signal
* **Expandable Section** = collapsed until opened
* **Hidden (Details)** = expert-only view

Your API returns **data completeness**.

So visually prioritize:

1. Identity
2. People
3. Condition
4. Opportunity
5. Details

NOT database categories.
